; РАСЧЁТ И ВЫВОД ПОКАЗАТЕЛЕЙ ЭФФЕКТИВНОСТИ
; ========================================================================
;
; После выполнения START анализируй результаты в отчёте GPSS:
;
; 1. QUEUES -> QREPAIR:
;    - AVE.CONT. = Q      (среднее число в очереди на ремонт, в любой момент времени)
;    - AVE.TIME  = W      (среднее время ожидания в очереди, ч)
;    - MAX CONT. =        (максимальная длина очереди за всё время моделирования)
;
; 2. STORAGES -> REPAIRMEN:
;    - CAP.      = C      (количество специалистов)
;    - AVE.C.    = U      (среднее число одновременно работающих специалистов)
;    - UTIL.     = ρ0     (коэффициент загрузки специалистов, [0...1])
;
; 3. STORAGES -> COMPUTERS:
;    - CAP.      = N      (количество компьютеров)
;    - AVE.C.    = n      (среднее число одновременно работающих компьютеров)
;    - UTIL.     = ρe     (коэффициент загрузки рабочих станций, [0...1])
;
; 4. Все TABLE (TABWAIT, TABSYS, TABQ, TABU и др.):
;    - Use Table Window для просмотра гистограмм распределений, 
;      например по времени ожидания, длине очереди, загрузке специалистов, 
;      длительности цикла работы каждого ПК и т.д.
;    - В Plot Window можешь строить графики изменения этих величин во времени, например:
;        Expression: Q$QREPAIR           (текущее число в очереди)
;        Expression: X$REPAIRMEN         (текущее число занятых специалистов)
;        Expression: X$COMPUTERS         (текущее число работающих ПК)
;        Expression: AVX$REPAIRMEN       (кумулятивная средняя загрузка специалистов)
;
; 5. Главные формулы для итоговых расчетов (заполни в Excel/Sheets):
;    - L   = Q + U                       (среднее число неисправных)
;    - n   = N - L                       (число исправных ПК)
;    - Tp  = W + t_о                     (среднее время простоя одного ПК)
;    - Tc  = t_но + Tp                   (среднее время полного цикла ПК)
;    - ρe  = t_но / Tc                   (доля времени ПК исправен)
;    - ρe/ρ0 = ρe / ρ0                   (коэффициент баланса системы)
;    - Y   = C * S_C + L * S             (убытки организации в час)
;
; Пример расчетов на основе симуляции:
;    Q = AVE.CONT. QREPAIR
;    U = AVE.C. REPAIRMEN
;    L = Q + U                     => полное число ПК простаивающих/в ремонте
;    n = N - L                     => сколько ПК реально работают
;    Tp = W + t_о                  => средний простой (ожидание + ремонт)
;    Tc = t_но + Tp                => средний цикл работы ПК
;    ρ0 = U / C                    => загруженность специалиста
;    ρe = t_но / Tc                => загруженность ПК
;    Y  = C*300 + L*500            => экономические убытки
;
; 6. Интерпретация графиков, которые можно построить:
;    - Q$QREPAIR        — динамика длины очереди по времени
;    - X$REPAIRMEN      — динамика числа занятых специалистов по времени
;    - AVX$COMPUTERS    — кумулятивная средняя загрузка ПК
;    - Все TABLE_*      — распределения (гистограмма) для соответствующих метрик
;
; Все показатели сравнивай для нескольких вариантов (C = 2, 3, 4) — вручную меняя STORAGE и анализируя выводы.
;
